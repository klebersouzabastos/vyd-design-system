# VYD — Internacionalização (i18n) & RTL

Como consumir o VYD em múltiplos idiomas e em escrita da direita-para-a-esquerda
(RTL: árabe, hebraico, persa). Introduzido na **v1.0.0**.

## Princípios

1. **O design system não traduz texto.** Rótulos, mensagens e formatos vêm do app
   (react-intl, i18next, `Intl.*`). O VYD entrega estrutura, tokens e componentes
   agnósticos de idioma.
2. **Direção é do documento, não do componente.** Defina uma vez, no topo:
   ```html
   <html lang="ar" dir="rtl">
   ```
   Todo o CSS do VYD que é sensível a direção usa **propriedades lógicas**
   (`border-inline-start`, `margin-inline-start`, `inset-inline-end`, `padding-inline-*`),
   então a UI espelha automaticamente quando `dir="rtl"`. Você não toca no CSS do VYD.

## O que já é RTL-safe

- **App shell** (`shell.css`): a barra de acento de 2px do item ativo (rail/ribbon-tab),
  as bordas de rails e inspector, e os divisores de grupo da ribbon usam
  `border-inline-*`. Em RTL, o left rail vira o *rail inicial* (à direita) e o
  inspector o *painel final* (à esquerda), sem código extra.
- **Primitivas** (`primitives.css`): faixa de acento de `alert`/`toast`, ícone e
  padding de `input-group`, `margin-inline-start:auto` de trailings (list, cmdk,
  alert close), asterisco de campo obrigatório — todos lógicos.

### Grid do shell em RTL

`grid-template-columns` é físico (esquerda→direita), mas as `grid-template-areas`
seguem a direção do container. Com `dir="rtl"` no `.vyd-app` (ou herdado do
`<html>`), as áreas `leftrail | canvas | rightpanel` reordenam para
`rightpanel | canvas | leftrail` visualmente. Nada a fazer.

## O que continua físico (de propósito)

- `.vyd-drawer--left` / `.vyd-drawer--right` — modificadores **explicitamente
  físicos**: "gaveta à direita" é uma escolha do produto, não do idioma. Se quiser
  comportamento lógico, use o lado que corresponde ao início/fim da sua leitura.
- Transforms de centralização (`translateX(-50%)`) — centralização é simétrica,
  não direcional.

## Números, unidades e dados tabulares

- **Sempre LTR**, mesmo em documento RTL. Números de engenharia, cotas, IDs, códigos
  e a coluna mono devem manter ordem ocidental. Envolva valores isolados:
  ```html
  <span dir="ltr" class="vyd-mono">P-12 · 3,20 m · Ø 25</span>
  ```
- A fonte mono (`--vyd-font-mono`) já aplica `tabular-nums` nos componentes de dados.
- Formate com `Intl.NumberFormat` / `Intl.DateTimeFormat` no app — o VYD não formata.

## Tipografia multilíngue

- `--vyd-font-sans` = Inter Tight → cai para `system-ui` em scripts não-latinos
  (árabe, CJK, cirílico). Para produção séria em árabe/hebraico, adicione uma fonte
  local no seu app **sem** alterar o token: defina `--vyd-font-sans` no escopo do
  documento traduzido:
  ```css
  [lang="ar"] { --vyd-font-sans: 'IBM Plex Sans Arabic', 'Inter Tight', sans-serif; }
  ```
  Isso mantém o restante do sistema intacto (você só reamarra um token semântico).
- A escala é compacta (base 13px); scripts com altura-x maior (árabe) respiram melhor
  subindo a densidade: `[lang="ar"] [data-vyd-density="comfortable"]`.

## Checklist de PR (apps)

- [ ] `<html dir>` correto para o locale.
- [ ] Números/IDs/mono envoltos em `dir="ltr"`.
- [ ] Ícones direcionais (setas de "próximo/voltar", chevrons de breadcrumb) espelhados
      em RTL — use os ícones lógicos ou `transform: scaleX(-1)` condicionado a `[dir=rtl]`.
- [ ] Testado com um locale RTL real (ex.: `ar`) e um locale longo (ex.: `de`) para
      caber texto que estica.

## Ícones direcionais

O sprite VYD tem ícones neutros. Para setas que devem apontar "para frente" na
direção da leitura, espelhe via CSS no seu app:

```css
[dir="rtl"] .vyd-icon--chevron-next { transform: scaleX(-1); }
```

Ícones de conteúdo (documento, camada, cubo da marca) **não** espelham.
